# Release Notes - v0.2.0

## Overview

This release introduces enhanced configuration flexibility for route file extensions, allowing greater control over which files are recognized as page entrypoints.

## üöÄ New Features

### Custom Entrypoint Extensions

- **New `entrypointExtensions` Option** - Added a new configuration property that allows you to specify custom file extensions for route entrypoints
- **Default Extensions** - Defaults to `[".tsx", ".jsx"]` for backward compatibility
- **Greater Flexibility** - Enables support for different file naming conventions in your routing directory

```typescript
reactToHtmlPlugin({
  shellPath: "src/shell.tsx",
  srcDir: "src/pages",
  entrypointExtensions: [".tsx", ".jsx", ".mdx"], // Custom extensions
});
```

## üìù Summary

Version 0.2.0 focuses on improving plugin modularity. The new `entrypointExtensions` option provides greater control over which file types are recognized as route entrypoints, enabling more flexible project structures and support for additional file formats like MDX.
